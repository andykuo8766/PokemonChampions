# OpenClaw 教學：如何辨別圖片中的寶可夢屬性

本文件用於教導 OpenClaw 在分析圖片時，自動辨識圖片中的寶可夢屬性（Type）。

---

# 目標

當使用者上傳圖片後：

1. 辨識圖片中的寶可夢
2. 判斷該寶可夢的屬性
3. 回傳標準格式結果

---

# 基本規則

## 一隻寶可夢最多有兩個屬性

範例：

- 皮卡丘 → 電
- 噴火龍 → 火 / 飛行
- 烈咬陸鯊 → 龍 / 地面

# 圖片辨識流程

## Step 1：辨識寶可夢名稱

先判斷圖片中的寶可夢是誰。

可依照以下資訊辨識：

- 身體顏色
- 臉部特徵
- 耳朵
- 翅膀
- 尾巴
- 四肢
- 花紋
- 官方配色
- Mega / 地區型態 / 悖謬種

---

## Step 2：確認型態

部分寶可夢不同型態會改變屬性。

必須注意：

### 地區型態

例如：

| 寶可夢 | 型態 | 屬性 |
|---|---|---|
| 九尾 | 一般 | 火 |
| 阿羅拉九尾 | 阿羅拉 | 冰 / 妖精 |

---

### Mega 進化

例如：

| 寶可夢 | 型態 | 屬性 |
|---|---|---|
| 噴火龍 | 普通 | 火 / 飛行 |
| Mega 噴火龍 X | Mega | 火 / 龍 |

---

### 太晶化

若圖片中有太晶皇冠：

- 優先判斷太晶屬性
- 同時保留原本屬性資訊

範例：

```json
{
  "pokemon": "班基拉斯",
  "original_type": ["岩石", "惡"],
  "tera_type": "飛行"
}
```
